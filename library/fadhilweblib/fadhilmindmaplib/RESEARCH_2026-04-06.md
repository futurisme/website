# fadhilmindmaplib research and design analysis (2026-04-06)

## Objective
Create a lightweight, high-speed mindmap framework branch under `fadhilweblib` that prioritizes runtime performance, deterministic behavior, extensibility, and full-stack deployability (frontend + backend + PostgreSQL).

## Repository-based research findings

### Existing fadhilweblib strengths
- Strong token/theme system (`--fwlb-*`) for visual consistency.
- Modular core + components architecture with reusable primitives.
- Existing performance/core namespace indicates optimization-first engineering fit.

### Mindmapmaker domain demands observed in repo
- Graph operations with hierarchical parent-child semantics.
- Deterministic layout for stable rendering and cacheability.
- Fast interactive editing with low allocation overhead.
- Durable snapshot persistence in PostgreSQL (Railway-compatible runtime).

## Architecture selected for fadhilmindmaplib

### 1) Struct-of-arrays graph store (`graph-store.ts`)
- `ids[]`, `titles[]` with typed arrays for `parents/depth/weights`.
- Dense id→index map for O(1) random access.
- Exponential capacity growth to reduce reallocation churn.

### 2) Deterministic layout engine (`layout-fast.ts`)
- Two-pass layered DFS:
  1. initial slot assignment
  2. parent centering from children span
- Linear complexity over tree-shaped maps.

### 3) Interaction runtime (`interaction-runtime.ts`)
- Microtask command batching (low frame overhead).
- Single patch emit per flush (observer efficiency).
- Bounded history ring for memory-safe undo support.

### 4) Syntax-first authoring (`syntax.ts`)
- Lightweight DSL parser for future “mindmap-as-code” workflows.
- Deterministic parse plan suitable for compile/cache pipelines.

### 5) Backend PostgreSQL support (`backend/postgres.ts`)
- `createPostgresMindmapRepository` for snapshot upsert + fetch.
- `postgresRailwaySchemaSql` for table/index bootstrap.
- Railway-focused env resolver (`resolveRailwayPostgresConfig`).

## Performance-focused implementation decisions
- Minimize object churn on hot paths.
- Typed arrays for numeric columns.
- Deterministic operations for stable render and test reproducibility.
- Zero hard dependency on a specific DB client (adapter accepts query executor).

## Future roadmap (recommended)
1. Incremental subtree layout recalculation.
2. Worker-thread/transferable buffer layout pipeline.
3. CRDT plugin bridge for collaborative sessions.
4. Postgres append-only event stream mode + snapshot compaction.
5. Auto-tuning heuristics for 10k+ node maps.

## Expected outcome
`fadhilmindmaplib` now includes a high-performance core, interaction runtime, syntax layer, and backend persistence foundation so future Mindmapmaker development can target this library as the primary framework path.
