# Mindmapmaker analysis (2026-04-06)

## Executive summary
Mindmapmaker is a substantial Next.js 14 application with a broad product surface (mind mapping, flow editing, collaboration, simulation/game systems, botmaker, and performance telemetry), backed by Prisma migrations and custom runtime modules. The project is not a simple landing page; it behaves as a platform with multiple experimental products under one router.

## Architecture inventory

### Framework and runtime
- Next.js `^14.0.0` with React 18 and TypeScript.
- Build pipeline performs `prisma generate`, then attempts `prisma migrate deploy`, then `next build`.
- Indicates production deployment assumes a database-capable environment and migration safety fallback.

### Routes (App Router)
Primary page routes found in `src/app`:
- `/` (landing)
- `/workspace`
- `/archive-lab`
- `/game`
- `/game-ideas`
- `/BotMaker`
- `/EngineTest`

### Feature domains
Feature modules indicate a multi-domain system:
- `flow` (visual graph/mind-map workspace + worker)
- `maps` (server + shared data handling)
- `collaboration` (presence)
- `botmaker` (server/shared schema)
- `game-ideas` (server/shared schema)
- `cpu-foundry`, `game-studio`, `gameplay` (simulation domains)
- `perf` (dashboard + ingest)
- `security` (scan/facial system modules)
- `workspace-home`

### Data model and migration history
Prisma migrations suggest active schema evolution:
- `0_init`
- map-id integer change
- perf metrics addition
- AI training runs addition
- training version/compression addition

## Operational implications
- **Complexity**: Cross-cutting concerns (editor, API, collaboration, analytics, game simulations) increase change risk and maintenance cost.
- **Deployment coupling**: Build behavior depends on database env vars; partial environments may produce non-blocking migration warnings.
- **Surface area**: Many public routes and APIs imply larger security, QA, and observability scope.
- **Version drift signal**: Mixed package versions (e.g., Next 14 runtime with newer analyzer package) should be periodically reviewed for compatibility.

## Current repository decision impact
In this change set, website exposure is intentionally reduced to static `/home` and `/portfolio` routes, while Mindmapmaker source remains in-repo for future reactivation, archival review, or selective extraction.

## Recommendation snapshot
1. Keep Mindmapmaker isolated behind non-public routing until a scoped relaunch plan exists.
2. If reactivated, split into bounded apps/domains (editor core vs. simulations vs. AI tooling).
3. Introduce CI gates for route-level smoke tests and migration safety checks.
4. Standardize dependency refresh cadence and compatibility matrix.
