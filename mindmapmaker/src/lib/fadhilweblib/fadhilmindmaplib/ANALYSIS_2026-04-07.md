# Analisa fadhilmindmaplib untuk engine `/mindmapmaker` (2026-04-07)

## Masalah utama (root cause)
1. **Duplikasi engine kritis di luar sublibrary**: logika jalur konektor cerdas sebelumnya hidup di `src/features/flow/flow-edge-routing-core.ts`, bukan di `fadhilmindmaplib`. Akibatnya, engine editor tidak benar-benar bergantung 100% pada sublibrary, dan evolusi performa/fitur sulit dipusatkan.

## Masalah lain yang menurunkan reliabilitas
1. **Boundary tidak jelas** antara domain UI (`features/flow`) dan domain engine (`fadhilmindmaplib`).
2. **Cache routing tidak memiliki batas eviksi** sebelumnya, sehingga berpotensi menumbuhkan memory footprint saat peta berubah terus-menerus.
3. **Konfigurasi routing tersebar** (toleransi/grid/gap) yang membuat tuning performa sulit dilakukan secara konsisten.

## Perbaikan yang diterapkan
1. **Transfer logika smart connector ke sublibrary**:
   - Ditambahkan modul `smart-connectors.ts` di `src/lib/fadhilweblib/fadhilmindmaplib` sebagai pusat logika:
     - klasifikasi orientasi jalur,
     - bus-routing untuk sibling,
     - lane offset deterministik,
     - normalisasi grid path.
2. **Abstraksi konfigurasi terpusat**:
   - Logika sublibrary menerima `SmartConnectorConfig`, sehingga tuning performa/layout bisa dipusatkan dan tidak hard-coded tersebar.
3. **Cache LRU ringan untuk routing**:
   - `createSmartConnectorCache(maxEntries)` ditambahkan untuk menjaga cache tetap bounded dan ramah browser ringan.
4. **Runtime fingerprint ultra-ringan**:
   - `createSmartConnectorRuntime(...)` menambahkan fast fingerprint berbasis urutan edge/node terkuantisasi grid.
   - Saat fingerprint sama, runtime mengembalikan geometri sebelumnya tanpa rebuild routing penuh.
5. **Integrasi balik ke flow workspace**:
   - `flow-edge-routing-core.ts` sekarang menjadi adapter tipis yang memanggil `fadhilmindmaplib`.
   - `flow-edge-routing.ts` memakai runtime baru agar jalur reroute saat drag/update menjadi lebih konsisten dan minim alokasi ulang.
   - Ini mengurangi duplikasi dan membuat alur evolusi fitur konektor fokus di sublibrary.
6. **Konektor masuk ke tengah body node (simetris)**:
   - Anchor source/target direvisi ke titik pusat node (bukan di outer box), sehingga hasil visual lebih rapi dan simetris.
   - Heuristik elbow horizontal diperbarui ke midpoint + adaptive clearance agar pemilihan rute lebih logis dan tidak membuat jalur melebar berlebihan.

## Dampak langsung
- Jalur konektor cerdas sekarang menjadi **single source of truth** di `fadhilmindmaplib`.
- Resiko divergensi antara engine dan UI berkurang signifikan.
- Kesiapan untuk optimasi lanjutan (worker vectorization, SIMD-like batching, heuristik drag-aware reroute) menjadi lebih tinggi karena fondasi sudah terpusat.
